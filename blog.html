Discuss the scope of var, let, and const


var: Has function scope. It is hoisted to the top of its function, meaning it can be used before it's declared. If declared outside any function, it becomes a global variable.
let: Has block scope, meaning it is limited to the block (enclosed by curly braces) where it's declared. It is not hoisted in a way that allows it to be used before the block.
const: Also has block scope, similar to let, but it additionally cannot be reassigned after declaration. However, the values they hold (if objects or arrays) can still be modified.
Understanding these scoping rules helps prevent unintended variable reassignments and contributes to more predictable code behavior.

Tell us the use cases of null and undefined

null is used to represent the intentional absence of any object value, while undefined typically indicates an
uninitialized or missing value in JavaScript.

What do you mean by REST API?

REST stands for Representational State Transfer, and it is an architectural style for designing networked applications. A
RESTful API (Application Programming Interface) is a set of rules and conventions for building and interacting with web
services that adhere to the principles of REST.
In a REST API:
1. Resources: Everything is treated as a resource, and each resource is uniquely identified by a Uniform Resource Identifier (URI). Resources can represent data entities or services.

2. Statelessness: Each request from a client to a server must contain all the information needed to understand and fulfill
that request. The server should not store any information about the client's state between requests.

3. Representation: Resources can have multiple representations, such as JSON or XML. Clients interact with these
representations to manipulate the state of resources.

4. Uniform Interface: The API should have a uniform and consistent interface, including standard conventions for naming
resources, using standard HTTP methods (GET, POST, PUT, DELETE), and returning standardized responses.
5. Stateless Communication: The communication between the client and server is stateless, meaning each request from a client
to a server must contain all the information needed to understand and fulfill that request.

RESTful APIs are widely used in web development and are the standard for building scalable and maintainable web services.
They rely on the HTTP protocol for communication, making them simple and widely adopted.
